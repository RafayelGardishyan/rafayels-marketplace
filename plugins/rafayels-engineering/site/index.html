01 — Workflow

The Feature Pipeline

Every feature follows the same disciplined path. Context first, then brainstorm, plan, implement, review, and document. Nothing ships incomplete.

01

Brainstorm

Explore approaches and trade-offs before writing code.

/workflows:brainstorm
02

Plan

File-level implementation plan with testing strategy.

/workflows:plan
03

Work

Isolated worktree execution with incremental commits.

/workflows:work
04

Review

16 agents analyze your PR in parallel across all domains.

/workflows:review
05

Learn & Improve

Document decisions so future work compounds, never repeats.

/workflows:compound
claude — ~/project
$
[brainstorm] Exploring 3 approaches: JWT, session-based, OAuth2...
[plan] Generated implementation plan — 12 files, 4 tests
[work] Working in worktree feature/auth...
[review] 16 agents analyzing PR #47...
[review] All checks passed. 0 critical, 2 suggestions.
✓ Feature shipped. ADR recorded. Knowledge compounded.
02 — Capabilities

Built for Serious Work

Not a toy. A complete engineering system with 32 agents, 27 skills, 31 commands, and native Pi extensions.

Multi-Agent Review

16 reviewers run in parallel: Go, TypeScript, Python, security, performance, architecture, data integrity, and more.

Architecture Decision Records

Semantic search and graph traversal of past decisions. Every architectural choice is recorded and connected.

Git Worktrees

Isolated development in worktrees. Main branch stays clean while features develop independently.

Knowledge Compounding

Solutions documentation means the same problem never needs solving twice. Institutional memory that persists.

Browser Testing

Automated visual testing with agent-browser. Navigate, interact, screenshot, and verify UI changes.

Dev Logs

Structured development logs capture what changed, why, and the context behind decisions.

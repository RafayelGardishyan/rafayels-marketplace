# Plan Detail Level Templates

Choose the appropriate detail level based on complexity and audience.

## MINIMAL (Quick Issue)

Best for: Simple bugs, small improvements, clear features.

Required sections:
- Problem/feature description (1 paragraph)
- Acceptance criteria (3-5 checkboxes)
- Context (any critical info)
- MVP code example (if applicable)

## MORE (Standard Issue)

Best for: Most features, complex bugs, team collaboration.

Required sections (adds to MINIMAL):
- Overview (comprehensive description)
- Problem Statement / Motivation
- Proposed Solution (high-level approach)
- Technical Considerations (architecture, performance, security)
- Acceptance Criteria (detailed, testable)
- Success Metrics
- Dependencies & Risks
- References & Research (file paths, URLs, PRs)

## A LOT (Comprehensive Issue)

Best for: Major features, architectural changes, complex integrations.

Required sections (adds to MORE):
- Executive summary
- Implementation Phases (Foundation → Core → Polish)
  - Each with: tasks, deliverables, success criteria, effort estimate
- Alternative Approaches Considered (and why rejected)
- Non-Functional Requirements (performance targets, security, a11y)
- Quality Gates (test coverage, docs, review approval)
- Risk Analysis & Mitigation
- Documentation Plan
- Internal + External References

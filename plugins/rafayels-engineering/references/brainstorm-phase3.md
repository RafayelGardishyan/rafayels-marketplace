# Brainstorm Phase 3: Capture the Design

## Document Template

Write the brainstorm document with this structure:

```markdown
---
date: YYYY-MM-DD
topic: <kebab-case-topic>
---

# <Topic Title>

## What We're Building
[Concise description — 1-2 paragraphs max]

## Why This Approach
[Brief explanation of approaches considered and why this one was chosen]

## Key Decisions
- [Decision 1]: [Rationale]
- [Decision 2]: [Rationale]

## Open Questions
- [Any unresolved questions for the planning phase]

## Next Steps
→ `/workflows:plan` for implementation details
```

## Guidelines

- Keep each section to 200-300 words maximum
- Focus on WHAT, not HOW (implementation details belong in the plan)
- State assumptions explicitly
- Flag open questions rather than making unvalidated decisions
- Reference similar existing features if they informed the approach
